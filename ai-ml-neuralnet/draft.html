1.1.1 What is Linear Algebra?

auto_storiesDefinition

Linear algebra is a branch of mathematics concerned with vectors, vector spaces, linear transformations, and systems of linear equations.

categoryCore Concepts

  • Vectors - Objects with magnitude and direction
  • Matrices - Rectangular arrays of numbers
  • Vector Spaces - Collections of vectors with operations
  • Linear Transformations - Functions preserving vector operations

psychologyKey Operations

  • Vector Addition - Combining vectors
  • Scalar Multiplication - Scaling vectors
  • Matrix Multiplication - Composing linear transformations
  • Determinants - Scaling factors of transformations

calculateMathematical Structures

  • Systems of Equations - Multiple linear relationships
  • Eigenvalues & Eigenvectors - Special vectors with simple transformations
  • Orthogonality - Perpendicularity in vector spaces
  • Subspaces - Smaller vector spaces within larger ones

appsApplications

  • Computer Graphics - 3D transformations
  • Machine Learning - Data representation
  • Engineering - Systems modeling
  • Physics - Quantum mechanics

1.1.2 Historical Context

history_eduEarly Foundations

200 BCE - 1600s

• Ancient Chinese and Babylonian methods for solving systems of linear equations
• Leibniz (1693): Early determinants and coefficient arrays
• Cramer (1750): Cramer's Rule for solving linear systems

psychologyFormal Development

1800s

• Gauss (1801): Gaussian elimination and least squares
• Cauchy (1812): Determinant properties and eigenvalue concepts
• Hamilton (1853): Quaternions and vector algebra
• Cayley (1858): Matrix algebra and notation

auto_awesomeAbstract Formulation

Late 1800s - Early 1900s

• Peano (1888): First axiomatic definition of vector spaces
• Hilbert (1906): Infinite-dimensional spaces
• Banach (1920s): Normed vector spaces
• Von Neumann (1929): Operator theory in quantum mechanics

trending_upEvolution of Linear Algebra

Linear algebra evolved from practical problem-solving (systems of equations) to abstract structures (vector spaces). This transition enabled applications in diverse fields from quantum physics to computer graphics, transforming it into a fundamental language of modern mathematics and science.

1.1.13 Mathematical Notation and Conventions

functionsVector Notation

  • Bold lowercase: v, w, x
  • Arrow notation: →v, →w
  • Components: v = (v₁, v₂, ..., vₙ)
  • Unit vectors: e₁, e₂, ..., eₙ
Example: v = (3, -2, 5) represents a vector in ℝ³

grid_onMatrix Notation

  • Bold uppercase: A, B, C
  • Elements: Aᵢⱼ (row i, column j)
  • Dimensions: A ∈ ℝᵐˣⁿ
  • Special matrices: I (identity), 0 (zero)
Example: A = [aᵢⱼ] where i = 1,...,m and j = 1,...,n

calculateOperations

  • Addition: A + B, v + w
  • Scalar multiplication: αv, cA
  • Matrix multiplication: AB, A·B
  • Dot product: v·w, ⟨v,w⟩
Example: (AB)ᵢⱼ = Σₖ AᵢₖBₖⱼ

auto_awesomeSpecial Functions

  • Norm: ||v|| (magnitude of vector)
  • Determinant: det(A), |A|
  • Trace: tr(A) (sum of diagonal)
  • Transpose: Aᵀ, A'
  • Inverse: A⁻¹
Example: ||v|| = √(v₁² + v₂² + ... + vₙ²)

lightbulbReading Mathematical Expressions

Mathematical notation is a precise language with specific conventions. When reading expressions, work from the inside out, paying attention to operator precedence. Practice translating between symbolic notation and verbal descriptions to strengthen your understanding.

1.1.14 Fundamental Concepts: Vectors

categoryDefinition & Representation

  • Mathematical Object - Magnitude and direction
  • Notation - Bold lowercase: v, w
  • Components - v = (v₁, v₂, ..., vₙ)
  • Examples - Physical quantities, data points
Example: v = (3, -2, 5) ∈ ℝ³

add_circleVector Operations

  • Addition - v + w = (v₁+w₁, v₂+w₂, ..., vₙ+wₙ)
  • Scalar Multiplication - αv = (αv₁, αv₂, ..., αvₙ)
  • Dot Product - v·w = v₁w₁ + v₂w₂ + ... + vₙwₙ
  • Cross Product - v × w (3D vectors only)
Example: 2v + 3w = (2v₁+3w₁, 2v₂+3w₂, 2v₃+3w₃)

straightenGeometric Interpretation

  • Magnitude - ||v|| = √(v₁² + v₂² + ... + vₙ²)
  • Direction - Unit vector: û = v/||v||
  • Angle - cos θ = (v·w)/(||v||·||w||)
  • Projection - projᵥw = ((v·w)/||v||²)v
Example: ||(3,4)|| = √(3²+4²) = 5

ruleKey Properties

  • Commutativity - v + w = w + v
  • Associativity - (u + v) + w = u + (v + w)
  • Distributivity - α(v + w) = αv + αw
  • Zero Vector - v + 0 = v
Example: v·w = w·v (commutative property)

visibilityVisual Representation

Vectors can be visualized as arrows in space, with length representing magnitude and direction indicating orientation. This geometric intuition helps understand operations like addition (parallelogram rule) and scalar multiplication (scaling).

1.1.15 Fundamental Concepts: Matrices

grid_onDefinition & Representation

  • Rectangular Array - Rows and columns of numbers
  • Notation - Bold uppercase: A, B
  • Elements - Aᵢⱼ (row i, column j)
  • Dimensions - A ∈ ℝᵐˣⁿ (m rows, n columns)
Example: A = [1 2; 3 4] is a 2×2 matrix

calculateMatrix Operations

  • Addition - A + B (element-wise)
  • Scalar Multiplication - αA (scale each element)
  • Matrix Multiplication - AB (row-by-column)
  • Transposition - Aᵀ (rows become columns)
Example: (AB)ᵢⱼ = Σₖ AᵢₖBₖⱼ

categorySpecial Matrices

  • Identity Matrix - I (diagonal = 1, others = 0)
  • Diagonal Matrix - Non-zero only on diagonal
  • Symmetric Matrix - A = Aᵀ
  • Zero Matrix - All elements = 0
Example: I₂ = [1 0; 0 1]

ruleKey Properties

  • Determinant - det(A), |A| (scaling factor)
  • Trace - tr(A) (sum of diagonal elements)
  • Rank - Number of linearly independent rows
  • Inverse - A⁻¹ where AA⁻¹ = I
Example: tr([1 2; 3 4]) = 1 + 4 = 5

visibilityVisual Representation

Matrices can represent linear transformations of space, mapping vectors to new vectors. The columns of a matrix show where the basis vectors are mapped, providing a powerful geometric interpretation of matrix operations.

1.1.16 Fundamental Concepts: Linear Transformations

transformDefinition & Properties

  • Function - T: V → W between vector spaces
  • Linearity - T(u+v) = T(u) + T(v)
  • Homogeneity - T(cu) = cT(u)
  • Notation - Often denoted by capital letters
Example: T([x,y]) = [2x, x+y] is a linear transformation

grid_onMatrix Representation

  • Standard Matrix - [T] where T(v) = [T]v
  • Columns - Images of basis vectors
  • Basis Change - [T]β→γ for different bases
  • Composition - [T∘S] = [T][S]
Example: Rotation by θ: [cosθ -sinθ; sinθ cosθ]

call_splitKernel & Range

  • Kernel (Null Space) - ker(T) = {v | T(v) = 0}
  • Range (Image) - range(T) = {T(v) | v ∈ V}
  • Rank-Nullity - dim(ker(T)) + dim(range(T)) = dim(V)
  • Injectivity - ker(T) = {0}
Example: For projection onto xy-plane, ker(T) = z-axis

3d_rotationGeometric Interpretation

  • Rotation - Preserves lengths and angles
  • Scaling - Changes size uniformly
  • Projection - Maps to lower-dimensional subspace
  • Shear - Preserves area but distorts shape
Example: T([x,y]) = [x,0] projects onto x-axis

visibilityVisualizing Transformations

Linear transformations can be visualized as distortions of space that preserve the grid structure. They map lines to lines and origin to origin, providing a powerful way to understand how matrices transform vector spaces.

1.2Advanced Vector Spaces - Theory

categoryVector Spaces & Subspaces

  • Vector Space: Set V with addition & scalar multiplication satisfying 10 axioms
  • Examples: ℝⁿ, ℂⁿ, Pₙ (polynomials), M_{m×n} (matrices)
  • Subspace: Non-empty subset W ⊆ V closed under addition & scalar multiplication
  • Subspace criterion: ∀ u,v ∈ W, ∀ c ∈ ℝ: u+v ∈ W and c·u ∈ W

linear_scaleLinear Independence & Basis

  • Linear Independence: c₁v₁ + ... + cₙvₙ = 0 only when c₁=...=cₙ=0
  • Span: span{v₁,...,vₙ} = {c₁v₁ + ... + cₙvₙ | cᵢ ∈ ℝ}
  • Basis: Linearly independent spanning set
  • Dimension: Number of vectors in any basis
  • Standard basis: e₁=(1,0,...,0), e₂=(0,1,...,0), ..., eₙ=(0,0,...,1)

transformLinear Transformations

  • Linear Transformation: T: V → W where T(u+v) = T(u) + T(v) and T(cu) = cT(u)
  • Kernel: ker(T) = {v ∈ V | T(v) = 0} (subspace of V)
  • Range: range(T) = {T(v) | v ∈ V} (subspace of W)
  • Matrix Representation: [T]_{β→γ} where β, γ are bases

square_footInner Product Spaces

  • Inner Product: Function ⟨·,·⟩: V×V→ℝ satisfying:
    • Symmetry: ⟨u,v⟩ = ⟨v,u⟩
    • Linearity: ⟨au+bv,w⟩ = a⟨u,w⟩ + b⟨v,w⟩
    • Positive-definiteness: ⟨v,v⟩ ≥ 0 and ⟨v,v⟩ = 0 ⇔ v=0
  • Orthogonality: u ⊥ v if ⟨u,v⟩ = 0
  • Norm: ||v|| = √⟨v,v⟩

lightbulbRank-Nullity Theorem

For any linear transformation T: V → W between finite-dimensional vector spaces:

dim(ker(T)) + dim(range(T)) = dim(V)

Equivalently, for any matrix A ∈ M_{m×n}:

nullity(A) + rank(A) = n

This theorem connects the dimensions of the kernel and range of a linear transformation, providing a fundamental relationship in linear algebra.

1.4Matrix Decompositions Deep Dive - QR Decomposition

descriptionTheoretical Foundations

  • Definition: A = QR where Q is orthogonal (QᵀQ = I) and R is upper triangular
  • Existence: Every matrix A ∈ ℝᵐˣⁿ with m ≥ n has a QR decomposition
  • Uniqueness: If A has full rank and R has positive diagonal entries, then QR decomposition is unique
  • Types: Full QR (Q ∈ ℝᵐˣᵐ, R ∈ ℝᵐˣⁿ) and Reduced QR (Q ∈ ℝᵐˣⁿ, R ∈ ℝⁿˣⁿ)

calculateComputation Methods

  • Gram-Schmidt Process: Orthogonalize columns of A to form Q
  • Householder Transformations: Reflect vectors through hyperplanes
    H = I - 2vvᵀ/||v||²
  • Givens Rotations: Rotate in coordinate planes to zero elements

auto_awesomeHouseholder Transformations

  • Reflection: Householder matrix reflects vectors about a hyperplane
  • Construction: For vector a, compute v = a ± sign(a₁)||a||e₁
  • Algorithm:
    for k = 1 to n
      x = A[k:m, k]
      v = sign(x₁)||x||e₁ + x
      v = v/||v||
      A[k:m, k:n] = A[k:m, k:n] - 2v(vᵀA[k:m, k:n])
      Qₖ = I - 2vvᵀ
    end
      Q = Q₁Q₂...Qₙ

app_settings_altApplications

  • Linear Systems: Solve Ax = b via QRx = b → Rx = Qᵀb
  • Least Squares: Minimize ||Ax - b||² by solving Rx = Qᵀb
  • Eigenvalue Computation: QR algorithm for finding eigenvalues
  • Orthogonalization: Creating orthonormal bases for subspaces
  • Numerical Stability: More stable than LU decomposition for ill-conditioned matrices

1.5Matrix Decompositions Deep Dive - SVD and Eigenvalue Decomposition

view_comfySingular Value Decomposition

  • Definition: A = UΣVᵀ
    U ∈ ℝᵐˣᵐ orthogonal, Σ ∈ ℝᵐˣⁿ diagonal, V ∈ ℝⁿˣⁿ orthogonal
  • Existence: Every matrix A ∈ ℝᵐˣⁿ has an SVD
  • Singular Values: Diagonal entries of Σ (σ₁ ≥ σ₂ ≥ ... ≥ 0)
  • Properties:
    • Rank = number of non-zero singular values
    • ||A||₂ = σ₁ (largest singular value)
    • Singular vectors form orthonormal bases

linear_scaleEigenvalue Decomposition

  • Definition: A = VΛV⁻¹
    V ∈ ℝⁿˣⁿ eigenvectors, Λ ∈ ℝⁿˣⁿ diagonal eigenvalues
  • Existence: Requires n linearly independent eigenvectors
  • Eigenvalues: Roots of characteristic polynomial det(A - λI) = 0
  • Properties:
    • Trace = sum of eigenvalues
    • Determinant = product of eigenvalues
    • Powers: Aᵏ = VΛᵏV⁻¹

compare_arrowsRelationship & Special Cases

  • Connection: For symmetric matrix A:
    • SVD: A = UΣVᵀ
    • EVD: A = QΛQᵀ
    • U = Q, V = Q, Σ = |Λ|
  • Symmetric Matrices: Always diagonalizable with orthogonal eigenvectors
  • Positive Definite: All eigenvalues positive, Cholesky decomposition
  • Normal Matrices: AAᵀ = AᵀA, unitarily diagonalizable

lightbulbSpectral Theorem

Every real symmetric matrix A can be diagonalized by an orthogonal matrix: A = QΛQᵀ where QᵀQ = I and Λ is diagonal.

appsGeometric Interpretations

  • SVD Geometry:
    • Vᵀ: Rotate/reflect input space
    • Σ: Scale along orthogonal axes
    • U: Rotate/reflect output space
  • EVD Geometry:
    • Eigenvectors: Principal axes of transformation
    • Eigenvalues: Scaling factors along these axes
  • Low-Rank Approximation:
    • Best rank-k approximation using largest singular values
    • Eckart-Young-Mirsky theorem: min||A-B||₂ = σₖ₊₁

1.7Advanced Matrix Calculus - Theory

functionsMatrix Derivatives

  • Scalar-by-Vector: ∂f/∂x where f: ℝⁿ → ℝ
    ∂f/∂x = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]ᵀ
  • Vector-by-Vector: ∂f/∂x where f: ℝⁿ → ℝᵐ
    [∂f/∂x]ᵢⱼ = ∂fᵢ/∂xⱼ
  • Scalar-by-Matrix: ∂f/∂X where f: ℝᵐˣⁿ → ℝ
    [∂f/∂X]ᵢⱼ = ∂f/∂Xᵢⱼ

trending_upGradients and Hessians

  • Gradient: Vector of first derivatives
    ∇f(x) = ∂f/∂x
  • Hessian: Matrix of second derivatives
    Hf(x) = ∂²f/∂x∂xᵀ
  • Properties:
    • Gradient points in direction of steepest ascent
    • Hessian symmetric for twice continuously differentiable functions
    • Positive definite Hessian ⇒ local minimum

sync_altMatrix Chain Rule & Product Rule

  • Chain Rule:
    ∂f(g(x))/∂x = (∂g/∂x)ᵀ ∂f/∂g
  • Product Rule:
    ∂(f(x)g(x))/∂x = f(x)∂g/∂x + g(x)∂f/∂x
  • Key Identities:
    • ∂(Ax)/∂x = Aᵀ
    • ∂(xᵀAx)/∂x = (A + Aᵀ)x
    • ∂(xᵀA)/∂x = A
    • ∂(AB)/∂X = A(∂B/∂X) + (∂A/∂X)B

speedOptimization in Linear Algebra

  • Gradient Descent: xₖ₊₁ = xₖ - α∇f(xₖ)
  • Newton's Method: xₖ₊₁ = xₖ - [Hf(xₖ)]⁻¹∇f(xₖ)
  • Conjugate Gradient: For quadratic systems Ax = b
  • Optimization Conditions:
    • First-order: ∇f(x*) = 0
    • Second-order: Hf(x*) ≽ 0

lightbulbTaylor's Theorem for Matrices

For a twice differentiable function f: ℝⁿ → ℝ:

f(x+p) = f(x) + pᵀ∇f(x) + ½pᵀHf(x+tp)p for some t ∈ (0,1)
