Chapter 1: Introduction to C

History and Evolution of C

C programming language was developed by Dennis Ritchie at Bell Laboratories in the early 1970s. It was created to write the UNIX operating system and has since become one of the most influential programming languages in history.

The evolution of C through various standards:

  • K&R C (1978): The original version described in "The C Programming Language" by Kernighan and Ritchie
  • ANSI C/C89 (1989): First standardized version by the American National Standards Institute
  • C90 (1990): ISO standard identical to ANSI C
  • C99 (1999): Added inline functions, variable-length arrays, single-line comments
  • C11 (2011): Added multithreading support, anonymous structures, Unicode support
  • C18 (2018): Minor update with bug fixes and clarifications
  • C23 (2023): Latest standard with new features and improvements

C's Place in Programming Ecosystem

C occupies a unique position in the programming world:

  • Foundation Language: Many modern languages (C++, Java, C#, Python) have syntax and concepts derived from C
  • System Programming: The language of choice for operating systems, embedded systems, and device drivers
  • Performance Critical: Used where speed and efficiency are paramount
  • Portability: C code can be compiled and run on virtually any platform
  • Low-level Access: Provides direct memory access and hardware interaction

Applications: System Software, Embedded, Game Dev, etc.

C is used across numerous domains:

Domain Applications Examples
System Software Operating Systems, Drivers Linux, Windows, macOS
Embedded Systems Microcontrollers, IoT Arduino, Raspberry Pi
Game Development Game Engines, Graphics Unreal Engine, OpenGL
Database Systems Database Management MySQL, PostgreSQL
Scientific Computing Numerical Analysis MATLAB, R
Networking Routers, Protocols TCP/IP stacks

Writing First Program (Hello World)

Let's write the classic "Hello, World!" program:

C 
#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}

Explanation:

  • #include <stdio.h> - Includes the standard input/output library
  • int main() - The main function where program execution begins
  • printf() - Function to print formatted output to the console
  • \n - Newline character
  • return 0; - Indicates successful program execution

Compilation Process: Preprocessor, Compiler, Linker

The compilation process involves several stages:

Bash 
# Compile the program
gcc hello.c -o hello

# Run the executable
./hello

Compilation Stages:

  1. Preprocessing: Handles preprocessor directives (#include, #define, etc.)
  2. Compilation: Converts C code to assembly language
  3. Assembly: Converts assembly code to machine code (object files)
  4. Linking: Combines object files and libraries to create an executable

Pro Tip

Use gcc -E hello.c to see preprocessor output, gcc -S hello.c to see assembly code, and gcc -c hello.c to generate object files.

Chapter 2: Basics & Syntax

Program Structure & Flow

Every C program follows a specific structure that the compiler expects. Understanding this structure is fundamental to writing correct C code.

A complete C program consists of:

  • Documentation Section: Comments about the program, author, date, etc.
  • Preprocessor Section: #include directives and #define macros
  • Global Declaration Section: Global variables and function prototypes
  • main() Function: The entry point of the program
  • User-defined Functions: Additional functions as needed
C 
/*
 * Program: Basic Structure Example
 * Author: Your Name
 * Date: Current Date
 */

#include <stdio.h>
#include <stdlib.h>

#define MAX_SIZE 100

// Global variable
int global_counter = 0;

// Function prototype
void increment_counter();

int main() {
    // Local variables
    int local_var = 5;
    
    printf("Global counter: %d\n", global_counter);
    printf("Local variable: %d\n", local_var);
    
    increment_counter();
    printf("After increment: %d\n", global_counter);
    
    return 0;
}

// Function definition
void increment_counter() {
    global_counter++;
}

Tokens, Keywords, and Identifiers

Tokens are the smallest individual units in a C program. There are five types of tokens:

Token Type Description Examples
Keywords Reserved words with special meaning int, char, if, else, while, for
Identifiers Names given to variables, functions, etc. main, printf, myVariable
Constants Fixed values that don't change 10, 3.14, 'A', "Hello"
Operators Symbols that perform operations +, -, *, /, =, ==, !=
Punctuators Symbols that separate or organize code ; , . ( ) [ ] { }

Keywords are reserved words that cannot be used as identifiers. C has 32 keywords:

int       long      register  return    short     signed    sizeof    static

Identifiers are names used to identify variables, functions, arrays, etc. Rules for identifiers:

  • Can contain letters, digits, and underscores
  • Must begin with a letter or underscore
  • Cannot be a keyword
  • Are case-sensitive (myVar and myvar are different)
  • No length limit, but only first 31 characters are significant

Naming Conventions

Follow these conventions for better code readability:

  • Use meaningful names (e.g., student_count instead of sc)
  • Use snake_case for variables and functions (my_variable)
  • Use UPPER_CASE for constants (MAX_SIZE)
  • Be consistent throughout your code

Input/Output: printf, scanf, gets, puts

C provides several functions for input and output operations. The most commonly used are printf() for output and scanf() for input.

printf() Function

The printf() function is used to print formatted output to the console.

int main() {
    int age = 25;
    float height = 5.9;
    char grade = 'A';
    char name[] = "John";

    printf("Name: %s\n", name);
    printf("Age: %d\n", age);
    printf("Height: %.1f\n", height);
    printf("Grade: %c\n", grade);
    
    // Multiple variables in one statement
    printf("%s is %d years old, %.1f feet tall, and got grade %c.\n", 
           name, age, height, grade);

}

scanf() Function

The scanf() function is used to read formatted input from the user.

C 
#include <stdio.h>

int main() {
    int age;
    float salary;
    char name[50];
    
    printf("Enter your name: ");
    scanf("%s", name);  // Note: No & for string arrays
    
    printf("Enter your age: ");
    scanf("%d", &age);  // & operator needed for non-array variables
    
    printf("Enter your salary: ");
    scanf("%f", &salary);
    
    printf("\n--- Your Information ---\n");
    printf("Name: %s\n", name);
    printf("Age: %d\n", age);
    printf("Salary: %.2f\n", salary);
    
    return 0;
}

gets() and puts() Functions

gets() reads a line from stdin, and puts() writes a line to stdout.

C 
#include <stdio.h>

int main() {
    char name[100];
    
    printf("Enter your full name: ");
    gets(name);  // Reads entire line including spaces
    
    printf("Hello, ");
    puts(name);  // Prints string with newline
    
    return 0;
}

Security Warning

gets() is dangerous and deprecated because it doesn't check buffer size. Use fgets() instead:

C 
char name[100];
fgets(name, sizeof(name), stdin);  // Safe alternative to gets()

Code Style & Formatting Best Practices

Good coding style makes your code more readable and maintainable. Here are some important conventions:

C 
// Good style example
#include <stdio.h>
#include <stdlib.h>

#define MAX_STUDENTS 100

// Function prototypes
int calculate_average(int scores[], int count);
void print_results(int average);

int main() {
    int student_scores[MAX_STUDENTS];
    int student_count = 0;
    int average_score;
    
    // Get input from user
    printf("Enter number of students: ");
    scanf("%d", &student_count);
    
    // Validate input
    if (student_count <= 0 || student_count > MAX_STUDENTS) {
        printf("Invalid number of students!\n");
        return 1;
    }
    
    // Calculate and display results
    average_score = calculate_average(student_scores, student_count);
    print_results(average_score);
    
    return 0;
}

// Function to calculate average
int calculate_average(int scores[], int count) {
    int sum = 0;
    for (int i = 0; i < count; i++) {
        sum += scores[i];
    }
    return sum / count;
}

// Function to print results
void print_results(int average) {
    printf("Average score: %d\n", average);
    if (average >= 90) {
        printf("Grade: A\n");
    } else if (average >= 80) {
        printf("Grade: B\n");
    } else if (average >= 70) {
        printf("Grade: C\n");
    } else {
        printf("Grade: F\n");
    }
}

Style Guidelines

  • Use consistent indentation (4 spaces recommended)
  • Place braces on the same line or next line consistently
  • Use meaningful variable and function names
  • Add spaces around operators for readability
  • Keep lines under 80-100 characters
  • Use blank lines to separate logical sections

Storage Classes: auto, register, static, extern

Storage classes define the scope, lifetime, and visibility of variables and functions.

Storage Class Scope Lifetime Default Value Storage Location
auto Local (block) Block Garbage Stack
register Local (block) Block Garbage Register (if available)
static Local (block) Program 0 Data segment
extern Global Program 0 Data segment
C 
#include <stdio.h>

// Global variable (extern by default)
int global_var = 100;

// Function prototype
void demonstrate_storage_classes();

int main() {
    // auto variable (default for local variables)
    auto int auto_var = 10;
    int auto_var2 = 20;  // auto is optional
    
    // register variable (request to store in register)
    register int counter = 0;
    
    printf("Global variable: %d\n", global_var);
    printf("Auto variables: %d, %d\n", auto_var, auto_var2);
    printf("Register variable: %d\n", counter);
    
    // Call function to demonstrate static
    demonstrate_storage_classes();
    demonstrate_storage_classes();
    demonstrate_storage_classes();
    
    return 0;
}

void demonstrate_storage_classes() {
    // static variable (retains value between function calls)
    static int static_var = 0;
    
    // auto variable (reinitialized each time)
    int auto_var = 0;
    
    static_var++;
    auto_var++;
    
    printf("Call %d: static_var = %d, auto_var = %d\n", 
           static_var, static_var, auto_var);
}

Type Conversion & Casting (Implicit & Explicit)

Type conversion is the process of converting one data type to another. C supports both implicit and explicit conversion.

Implicit Type Conversion

C 
#include <stdio.h>

int main() {
    int int_var = 10;
    float float_var = 3.14f;
    double double_var = 2.71828;
    char char_var = 'A';
    
    // Implicit conversion examples
    float result1 = int_var + float_var;     // int converted to float
    double result2 = int_var + double_var;   // int converted to double
    int result3 = int_var + char_var;        // char converted to int
    
    printf("int + float = %.2f\n", result1);
    printf("int + double = %.2f\n", result2);
    printf("int + char = %d\n", result3);
    
    // Type promotion in expressions
    char c1 = 50, c2 = 60;
    int result4 = c1 + c2;  // chars promoted to int before addition
    printf("char + char = %d\n", result4);
    
    return 0;
}

Explicit Type Conversion (Casting)

C 
#include <stdio.h>

int main() {
    int int_var = 10;
    float float_var = 3.14f;
    double double_var = 2.71828;
    
    // Explicit casting examples
    int result1 = (int)float_var;           // float to int (truncation)
    float result2 = (float)int_var;         // int to float
    double result3 = (double)int_var;       // int to double
    char result4 = (char)int_var;           // int to char
    
    printf("float to int: %d\n", result1);
    printf("int to float: %.2f\n", result2);
    printf("int to double: %.2f\n", result3);
    printf("int to char: %c\n", result4);
    
    // Practical example: division
    int a = 5, b = 2;
    float result5 = (float)a / b;           // Cast to float for precise division
    printf("5 / 2 = %.2f\n", result5);
    
    return 0;
}

Type Conversion Pitfalls

  • Data Loss: Converting from larger to smaller types can lose data
  • Sign Extension: Converting signed to unsigned can cause unexpected results
  • Precision Loss: Floating-point to integer conversion truncates decimal part
  • Overflow: Large values may overflow when converted to smaller types

Arithmetic Operators

Arithmetic operators perform mathematical operations on numeric operands.

Operator Description Example Result
+ Addition a + b Sum of a and b

* Multiplication a * b Product of a and b
/ Division a / b Quotient of a and b

C 
#include <stdio.h>

int main() {
    int a = 10, b = 3;
    float x = 10.0f, y = 3.0f;
    
    printf("Integer arithmetic:\n");

    printf("x + y = %.2f\n", x + y);    // 13.00
    printf("x - y = %.2f\n", x - y);    // 7.00
    printf("x * y = %.2f\n", x * y);    // 30.00
    printf("x / y = %.2f\n", x / y);    // 3.33 (precise division)
    
    // Mixed arithmetic
    printf("\nMixed arithmetic:\n");
    printf("a + y = %.2f\n", a + y);    // 13.00 (int promoted to float)
    printf("x / b = %.2f\n", x / b);    // 3.33 (int promoted to float)
    
    return 0;
}

Relational and Logical Operators

Relational operators compare values and return boolean results. Logical operators combine boolean expressions.

Relational Operators

Operator Description Example Result (if a=5, b=3)
== Equal to a == b 0 (false)
!= Not equal to a != b 1 (true)
> Greater than a > b 1 (true)
< Less than a < b 0 (false)
>= Greater than or equal to a >= b 1 (true)
<= Less than or equal to a <= b 0 (false)

Logical Operators

Operator Description Example Result
&& Logical AND a && b 1 if both a and b are true
|| Logical OR a || b 1 if either a or b is true
! Logical NOT !a 1 if a is false, 0 if a is true
C 
#include <stdio.h>

int main() {
    int a = 5, b = 3, c = 5;
    
    printf("Relational operators:\n");
    printf("a == b: %d\n", a == b);     // 0 (false)
    printf("a != b: %d\n", a != b);     // 1 (true)
    printf("a > b: %d\n", a > b);       // 1 (true)
    printf("a < b: %d\n", a < b);       // 0 (false)
    printf("a >= b: %d\n", a >= b);     // 1 (true)
    printf("a <= b: %d\n", a <= b);     // 0 (false)
    printf("a == c: %d\n", a == c);     // 1 (true)
    
    printf("\nLogical operators:\n");
    printf("(a > b) && (a == c): %d\n", (a > b) && (a == c));  // 1 (true)
    printf("(a > b) || (a < b): %d\n", (a > b) || (a < b));    // 1 (true)
    printf("!(a == b): %d\n", !(a == b));                     // 1 (true)
    printf("(a > b) && (a < b): %d\n", (a > b) && (a < b));    // 0 (false)
    
    // Short-circuit evaluation
    int x = 5, y = 0;
    printf("\nShort-circuit evaluation:\n");
    printf("(x > 0) && (y != 0): %d\n", (x > 0) && (y != 0));  // 0 (false, second part not evaluated)
    printf("(x < 0) || (y == 0): %d\n", (x < 0) || (y == 0));  // 1 (true, second part not evaluated)
    
    return 0;
}

Operator Precedence & Associativity

Operator precedence determines the order in which operations are performed. Associativity determines the order when operators have the same precedence.

3 * / % Multiplicative Left to right
4 + - Additive Left to right
5 << >> Shift Left to right
6 < <= > >= Relational Left to right
7 == != Equality Left to right
8 & Bitwise AND Left to right
9 ^ Bitwise XOR Left to right
10 | Bitwise OR Left to right

12 || Logical OR Left to right

#include <stdio.h>

int main() {
    int a = 5, b = 10, c = 15, d = 20;
    
    // Precedence examples
    printf("Precedence examples:\n");
    
    // Multiplication has higher precedence than addition
    int result1 = a + b * c;  // b * c first, then + a
    printf("a + b * c = %d\n", result1);    // 5 + 150 = 155
    
    // Parentheses change precedence
    int result2 = (a + b) * c;  // a + b first, then * c
    printf("(a + b) * c = %d\n", result2);  // 15 * 15 = 225
    
    // Relational operators have higher precedence than logical operators
    int result3 = a < b && c < d;  // Both comparisons first, then &&
    printf("a < b && c < d = %d\n", result3);  // 1 && 1 = 1
    
    // Assignment has lower precedence than arithmetic
    int result4 = a += b * c;  // b * c first, then +=
    printf("a += b * c = %d\n", result4);  // a = 5 + 150 = 155
    
    // Associativity examples
    printf("\nAssociativity examples:\n");
    
    // Left associativity: same precedence operators evaluated left to right
    int result5 = a - b + c;  // (a - b) + c
    printf("a - b + c = %d\n", result5);  // (155 - 10) + 15 = 160
    
    // Right associativity: assignment operators
    int x, y, z;
    x = y = z = 100;  // z = 100, y = z, x = y
    printf("x = y = z = 100: x=%d, y=%d, z=%d\n", x, y, z);
    
    // Complex expression
    int result6 = a + b * c - d / 2 + (a > b ? a : b);
    printf("Complex: a + b * c - d / 2 + (a > b ? a : b) = %d\n", result6);
    
    return 0;
}

Expression Evaluation & Order

Understanding how expressions are evaluated is crucial for writing correct code. C follows specific rules for expression evaluation.

C 
#include <stdio.h>

int main() {
    int a = 5, b = 10, c = 15;
    
    // Expression evaluation order
    printf("Expression evaluation order:\n");
    
    // Function calls are evaluated before the function is called
    printf("Function call evaluation:\n");
    int func1() { printf("func1 called\n"); return 1; }
    int func2() { printf("func2 called\n"); return 2; }
    int result1 = func1() + func2();
    printf("Result: %d\n", result1);
    
    // Side effects and sequence points
    printf("\nSide effects and sequence points:\n");
    int x = 5;
    printf("x = %d\n", x);
    
    // Undefined behavior: multiple modifications between sequence points
    // int y = x++ + x++;  // Undefined behavior - don't do this!
    
    // Safe approach: use separate statements
    x = 5;
    int temp1 = x++;
    int temp2 = x++;
    int y = temp1 + temp2;
    printf("Safe increment: x = %d, y = %d\n", x, y);
    
    // Short-circuit evaluation
    printf("\nShort-circuit evaluation:\n");
    int p = 1, q = 0;
    int result2 = p && q++;  // q++ is not evaluated because p is false
    printf("p && q++: result = %d, q = %d\n", result2, q);
    
    q = 0;
    int result3 = p || q++;  // q++ is not evaluated because p is true
    printf("p || q++: result = %d, q = %d\n", result3, q);
    
    // Comma operator
    printf("\nComma operator:\n");
    int i = (a = 1, b = 2, c = 3);  // a=1, b=2, c=3, i=3
    printf("i = (a = 1, b = 2, c = 3): i=%d, a=%d, b=%d, c=%d\n", i, a, b, c);
    
    return 0;
}

Important: Undefined Behavior

Some expressions have undefined behavior in C:

  • Modifying a variable multiple times between sequence points
  • Reading and modifying the same variable without sequence point
  • Function calls with overlapping arguments
  • Always use parentheses to make your intent clear

Short-circuiting Logical Evaluation

C uses short-circuit evaluation for logical operators, which can be both useful and dangerous.

C 
#include <stdio.h>

    int a = 5, b = 0;
    
    // Short-circuit with && (AND)
    printf("Short-circuit with &&:\n");
    printf("a = %d, b = %d\n", a, b);
    
    // In (a > 0) && (b != 0), if first part is false, second is not evaluated
    int result1 = (a > 0) && (b != 0);

    printf("b is still %d (not evaluated)\n", b);

    printf("\nShort-circuit with ||:\n");
    // In (a > 0) || (b != 0), if first part is true, second is not evaluated
    int result2 = (a > 0) || (b != 0);
    printf("(a > 0) || (b != 0) = %d\n", result2);
    printf("b is still %d (not evaluated)\n", b);
    
    // Practical use: avoiding division by zero
    printf("\nAvoiding division by zero:\n");
    int x = 10, y = 0;
    
    if (y != 0 && x / y > 2) {
        printf("x/y > 2\n");
    } else {
        printf("Cannot divide or condition not met\n");
    }
    
    // Safe pointer access
    printf("\nSafe pointer access:\n");
    int *ptr = NULL;
    int value = 42;
    
    if (ptr != NULL && *ptr > 0) {
        printf("Pointer points to positive value\n");
    } else {
        printf("Pointer is NULL or points to non-positive value\n");
    }
    
    // Function calls with short-circuit
    printf("\nFunction calls with short-circuit:\n");
    int func1() { printf("func1 called\n"); return 0; }
    int func2() { printf("func2 called\n"); return 1; }

Understanding how parameters are passed to functions is essential for effective programming.

Recursive Functions

Recursive functions call themselves to solve problems by breaking them down into smaller subproblems.

C 
#include <stdio.h>

// Factorial using recursion
int factorial(int n) {
    if (n <= 1) {
        return 1;  // Base case
    } else {
        return n * factorial(n - 1);  // Recursive case
    }
}

// Fibonacci sequence using recursion
int fibonacci(int n) {
    if (n <= 1) {
        return n;  // Base case
    } else {
        return fibonacci(n - 1) + fibonacci(n - 2);  // Recursive case
    }
}

// Power function using recursion
double power(double base, int exp) {
    if (exp == 0) {
        return 1.0;  // Base case
    } else if (exp > 0) {
        return base * power(base, exp - 1);  // Recursive case (positive exponent)
    } else {
        return 1.0 / power(base, -exp);  // Recursive case (negative exponent)
    }
}

// Binary search using recursion
int binary_search(int arr[], int left, int right, int target) {
    if (left > right) {
        return -1;  // Base case: not found
    }
    
    int mid = left + (right - left) / 2;
    
    if (arr[mid] == target) {
        return mid;  // Base case: found
    } else if (arr[mid] < target) {
        return binary_search(arr, mid + 1, right, target);  // Search right half
    } else {
        return binary_search(arr, left, mid - 1, target);  // Search left half
    }
}

int main() {
    // Testing factorial
    printf("Factorial of 5: %d\n", factorial(5));
    printf("Factorial of 7: %d\n", factorial(7));
    
    // Testing Fibonacci
    printf("\nFibonacci sequence (first 10 terms):\n");
    for (int i = 0; i < 10; i++) {
        printf("%d ", fibonacci(i));
    }
    printf("\n");
    
    // Testing power function
    printf("\nPower calculations:\n");
    printf("2^3 = %.0f\n", power(2.0, 3));
    printf("3^4 = %.0f\n", power(3.0, 4));
    printf("2^-2 = %.2f\n", power(2.0, -2));
    
    // Testing binary search
    int arr[] = {1, 3, 5, 7, 9, 11, 13, 15};
    int target = 7;
    int result = binary_search(arr, 0, 7, target);
    printf("\nBinary search for %d: ", target);
    if (result != -1) {
        printf("Found at index %d\n", result);
    } else {
        printf("Not found\n");
    }
    
    return 0;
}
