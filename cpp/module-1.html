1.1 Introduction to C++ Beginner

▼

What is C++?

C++ is a powerful general-purpose programming language created by Bjarne Stroustrup at Bell Labs in 1979. It was developed as an extension of the C programming language, originally named "C with Classes" before being renamed C++ in 1983.

Key Concept: The Evolution of C++

C++ has evolved through several major standards, each adding new features and improvements:

  • C++98 (1998): First standardized version
  • C++03 (2003): Minor bug fixes
  • C++11 (2011): Major update with modern features
  • C++14 (2014): Incremental improvements
  • C++17 (2017): More features and library updates
  • C++20 (2020): Concepts, modules, coroutines
  • C++23 (2023): Latest standard with new features

First C++ Program - "Hello, World!"

#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}
Terminal
$ g++ hello.cpp -o hello
$ ./hello
Hello, World!
$

Understanding the "Hello, World!" Program

  • #include <iostream> - Includes the input/output stream library
  • int main() - The main function where program execution begins
  • std::cout - Standard character output stream (usually the console)
  • << - Stream insertion operator
  • std::endl - Inserts a newline and flushes the output buffer
  • return 0; - Indicates successful program execution

Why Learn C++?

C++ remains one of the most popular programming languages for several reasons:

Key Advantages of C++

  • Performance: C++ offers near-hardware level performance, making it ideal for system programming, game development, and high-performance applications.
  • Control: Provides fine-grained control over system resources, especially memory management.
  • Object-Oriented Programming: Supports OOP principles like encapsulation, inheritance, and polymorphism.
  • Generic Programming: Templates allow writing code that works with different data types.
  • Large Ecosystem: Extensive libraries, frameworks, and tools available.
  • Industry Standard: Widely used in industries like gaming, finance, embedded systems, and high-frequency trading.

Real-World Applications of C++

  • Operating Systems: Parts of Windows, macOS, and Linux are written in C++
  • Game Development: Unreal Engine, Unity (parts), and many game studios use C++
  • Web Browsers: Chrome, Firefox, and Safari use C++ for performance-critical components
  • Databases: MySQL, MongoDB, and PostgreSQL are built with C++
  • Scientific Computing: Used in simulations, data analysis, and research
  • Embedded Systems: Automotive systems, IoT devices, and industrial controllers
  • Financial Systems: High-frequency trading platforms and banking systems

C++ Programming Paradigms

C++ is a multi-paradigm language, supporting several programming styles:

Programming Paradigms in C++

  • Procedural Programming: C++ retains all features of C, allowing procedural programming with functions and structured code.
  • Object-Oriented Programming (OOP): C++ was originally designed to add OOP features to C, including classes, objects, inheritance, and polymorphism.
  • Generic Programming: Templates allow writing code that works with multiple data types without being rewritten for each type.
  • Functional Programming: Modern C++ (C++11 and later) includes features like lambda expressions that support functional programming concepts.
  • Metaprogramming: Template metaprogramming allows computation at compile-time, enabling powerful optimizations and code generation.

Example: Different Paradigms in C++

#include <iostream>
#include <vector>
#include <algorithm>
#include <functional>

// Procedural approach
int add_procedural(int a, int b) {
    return a + b;
}

// Object-Oriented approach
class Calculator {
public:
    int add(int a, int b) {
        return a + b;
    }
};

// Generic approach
template<typename T>
T add_generic(T a, T b) {
    return a + b;
}

// Functional approach (C++11)
int main() {
    // Procedural
    int result1 = add_procedural(5, 3);
    
    // Object-Oriented
    Calculator calc;
    int result2 = calc.add(5, 3);
    
    // Generic
    int result3 = add_generic(5, 3);
    double result4 = add_generic(5.5, 3.3);
    
    // Functional with lambda
    auto add_lambda = [](int a, int b) { return a + b; };
    int result5 = add_lambda(5, 3);
    
    std::cout << "Results: " << result1 << ", " << result2 << ", " 
              << result3 << ", " << result4 << ", " << result5 << std::endl;
    
    return 0;
}

C++ vs Other Languages

Understanding how C++ compares to other popular programming languages helps in choosing the right tool for the job.

C++ vs C

  • C++ is a superset of C (with minor exceptions)
  • C++ adds object-oriented features not present in C
  • C++ includes exception handling, templates, and other modern features
  • C++ has stronger type checking than C
  • C code can generally be compiled with a C++ compiler

C++ vs Java

  • C++ is compiled to machine code; Java is compiled to bytecode that runs on a virtual machine
  • C++ allows manual memory management; Java uses garbage collection
  • C++ supports multiple inheritance; Java supports single inheritance with interfaces
  • C++ has templates; Java has generics
  • C++ generally offers better performance; Java offers better portability

C++ vs Python

  • C++ is statically typed; Python is dynamically typed
  • C++ is compiled; Python is interpreted
  • C++ requires explicit memory management; Python uses garbage collection
  • C++ is much faster for computation-intensive tasks
  • Python is generally easier to learn and write
  • C++ is used for performance-critical code; Python is often used for scripting and rapid development

When to Choose C++

Choose C++ when you need:

  • Maximum performance and efficiency
  • Direct hardware access and low-level control
  • Object-oriented design with performance
  • System programming or embedded development
  • Game development or graphics programming
  • High-frequency trading or financial applications

1.2 Basic Program Structure Beginner

▼

Anatomy of a C++ Program

Every C++ program follows a basic structure that consists of several key components. Understanding these components is essential for writing and reading C++ code.

Basic Components of a C++ Program

  1. Preprocessor Directives: Instructions processed before compilation (e.g., #include)
  2. Namespaces: Used to avoid naming conflicts (e.g., using namespace std;)
  3. Function Declarations/Definitions: Blocks of code that perform specific tasks
  4. Variables: Named storage locations for data
  5. Statements: Instructions that perform actions
  6. Expressions: Combinations of values, variables, and operators
  7. Comments: Explanatory text ignored by the compiler

Complete Program Structure Example

// Preprocessor directive to include input/output stream library
#include <iostream>

// Using directive to avoid typing std:: before standard library features
using namespace std;

// Function declaration
void greet();

// Global variable (can be accessed by any function)
int globalVar = 100;

// Main function - program entry point
int main() {
    // Local variable (only accessible within main)
    int localVar = 50;
    
    // Output statement
    cout << "Global variable: " << globalVar << endl;
    cout << "Local variable: " << localVar << endl;
    
    // Function call
    greet();
    
    // Return statement indicating successful execution
    return 0;
}

// Function definition
void greet() {
    cout << "Hello from the greet function!" << endl;
}

Preprocessor Directives

Preprocessor directives are instructions that are processed before the actual compilation of the code begins. They all start with a hash symbol (#).

Common Preprocessor Directives

  • #include: Includes the contents of a file (header or source)
  • #define: Defines a macro or symbolic constant
  • #undef: Undefines a macro
  • #ifdef: Checks if a macro is defined
  • #ifndef: Checks if a macro is not defined
  • #endif: Ends a conditional block
  • #if: Conditional compilation
  • #else: Alternative for #if
  • #elif: Else if for #if

Preprocessor Directives Examples

// Including standard library headers
#include <iostream>     // Input/output stream
#include <string>       // String manipulation
#include <vector>       // Dynamic array container

// Including custom headers
#include "myheader.h"    // Custom header file

// Defining constants
#define PI 3.14159
#define MAX_SIZE 100

// Conditional compilation
#define DEBUG_MODE 1

#ifdef DEBUG_MODE
    cout << "Debug mode is enabled" << endl;
#endif

// Header guards to prevent multiple inclusion
#ifndef MYHEADER_H
#define MYHEADER_H

// Header content goes here

#endif // MYHEADER_H

Header Guards

Header guards are essential to prevent multiple inclusion of the same header file, which can cause compilation errors due to redefinition of types, functions, or variables. The pattern #ifndef ... #define ... #endif is the traditional way to implement header guards.

Namespaces

Namespaces are used to organize code into logical groups and to prevent name collisions that can occur especially when your code base includes multiple libraries.

The std Namespace

The C++ standard library is defined in the std namespace. When you use standard library features, you need to either:

  1. Prefix them with std::
  2. Use a using declaration for specific items
  3. Use a using namespace std; directive (not recommended in headers)

Namespace Examples

#include <iostream>

// Method 1: Using std:: prefix
int main() {
    std::cout << "Hello with std:: prefix" << std::endl;
    return 0;
}

// Method 2: Using declaration
#include <iostream>
using std::cout;
using std::endl;

int main() {
    cout << "Hello with using declaration" << endl;
    return 0;
}

// Method 3: Using directive (not recommended for headers)
#include <iostream>
using namespace std;

int main() {
    cout << "Hello with using directive" << endl;
    return 0;
}

// Creating custom namespaces
namespace Math {
    const double PI = 3.14159;
    double add(double a, double b) {
        return a + b;
    }
}

namespace Physics {
    const double GRAVITY = 9.81;
    double calculateForce(double mass, double acceleration) {
        return mass * acceleration;
    }
}

int main() {
    // Using custom namespaces
    double sum = Math::add(5.5, 3.3);
    double force = Physics::calculateForce(10.0, 2.0);
    
    std::cout << "Sum: " << sum << std::endl;
    std::cout << "Force: " << force << std::endl;
    
    return 0;
}

Best Practices for Namespaces

  • In header files, always use the explicit std:: prefix to avoid namespace pollution
  • In implementation files, you can use using declarations for frequently used standard library features
  • Avoid using namespace std; in header files as it affects all files that include the header
  • Create your own namespaces for your libraries to avoid conflicts with other code

The main() Function

The main() function is the entry point of every C++ program. When the operating system runs your program, it starts by calling the main() function.

main() Function Signatures

The main() function can have two valid signatures:

  1. int main() - No parameters
  2. int main(int argc, char* argv[]) - With command-line arguments

In both cases, the function returns an int value to the operating system:

  • 0 or EXIT_SUCCESS: Program executed successfully
  • Non-zero value or EXIT_FAILURE: Program encountered an error

main() Function Examples

#include <iostream>

// Simple main function
int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;  // Indicates successful execution
}

// main() function with command-line arguments
#include <iostream>
#include <string>

int main(int argc, char* argv[]) {
    std::cout << "Number of arguments: " << argc << std::endl;
    
    for (int i = 0; i < argc; ++i) {
        std::cout << "Argument " << i << ": " << argv[i] << std::endl;
    }
    
    return 0;
}

// Using EXIT_SUCCESS and EXIT_FAILURE
#include <iostream>
#include <cstdlib>  // For EXIT_SUCCESS and EXIT_FAILURE

int main(int argc, char* argv[]) {
    if (argc < 2) {
        std::cerr << "Error: Missing argument" << std::endl;
        return EXIT_FAILURE;
    }
    
    std::cout << "Processing: " << argv[1] << std::endl;
    
    return EXIT_SUCCESS;
}
Terminal
$ g++ program.cpp -o program
$ ./program arg1 arg2 arg3
Number of arguments: 4
Argument 0: ./program
Argument 1: arg1
Argument 2: arg2
Argument 3: arg3
$

Comments

Comments are explanatory notes in the source code that are ignored by the compiler. They are essential for making code readable and maintainable.

Types of Comments in C++

  • Single-line comments: Start with // and continue to the end of the line
  • Multi-line comments: Start with /* and end with */

Comment Examples

#include <iostream>

// This is a single-line comment

/*
 * This is a multi-line comment.
 * It can span multiple lines.
 */

/**
 * This is a documentation comment.
 * Often used to document functions and classes.
 * Some tools can extract these to generate documentation.
 */

int main() {
    // Variable declaration with comment
    int age = 25;  // Age in years
    
    /* 
     * This is a block comment
     * inside a function
     */
    std::cout << "Age: " << age << std::endl;
    
    return 0;
}

Best Practices for Comments

  • Comment the "why" rather than the "what" - focus on explaining the reasoning behind the code
  • Keep comments up-to-date with the code - outdated comments can be misleading
  • Use clear and concise language
  • Avoid obvious comments like "x = 5; // Assign 5 to x"
  • Use documentation comments for public APIs to generate documentation

Introduction to I/O Streams

C++ uses streams for input and output operations. A stream is an abstraction that represents a sequence of bytes flowing into or out of your program. The <iostream> library provides the basic I/O functionality.

Standard Stream Objects

  • std::cout: Standard character output stream (usually the console)
  • std::cin: Standard character input stream (usually the keyboard)
  • std::cerr: Standard error stream for error messages (unbuffered)
  • std::clog: Standard logging stream for logging messages (buffered)

Basic Output with cout

#include <iostream>

int main() {
    // Outputting a string
    std::cout << "Hello, World!" << std::endl;
    
    // Outputting multiple values
    int age = 25;
    std::cout << "Age: " << age << std::endl;
    
    // Outputting without a newline
    std::cout << "This is on the same line. ";
    std::cout << "This continues the line." << std::endl;
    
    // Using different newline methods
    std::cout << "Method 1: Using std::endl" << std::endl;
    std::cout << "Method 2: Using \\n" << "\n";
    
    // Outputting special characters
    std::cout << "Tab:\tSeparated\tValues" << std::endl;
    std::cout << "Quote: \"Hello\"" << std::endl;
    std::cout << "Backslash: \\" << std::endl;
    
    return 0;
}

std::endl vs \n

  • std::endl: Inserts a newline character and flushes the output buffer
  • \n: Inserts a newline character but doesn't flush the buffer
  • Flushing the buffer ensures that the output is immediately written to the console, but it can impact performance
  • For performance-critical code, use \n instead of std::endl

Input with cin

The std::cin object is used to read input from the standard input device (usually the keyboard). It uses the extraction operator (>>) to read values into variables.

Basic Input Operations

  • std::cin >> variable;: Reads a value into the variable
  • std::cin skips whitespace (spaces, tabs, newlines) by default
  • Input operations can be chained to read multiple values
  • std::cin has a boolean value that can be checked to see if the input was successful

Basic Input Examples

#include <iostream>
#include <string>

int main() {
    // Reading an integer
    int age;
    std::cout << "Enter your age: ";
    std::cin >> age;
    std::cout << "You entered: " << age << std::endl;
    
    // Reading multiple values
    int x, y;
    std::cout << "Enter two numbers separated by space: ";
    std::cin >> x >> y;
    std::cout << "Sum: " << x + y << std::endl;
    
    // Reading a string (only reads until first whitespace)
    std::string name;
    std::cout << "Enter your first name: ";
    std::cin >> name;
    std::cout << "Hello, " << name << "!" << std::endl;
    
    // Clearing the input buffer
    std::cin.ignore();  // Ignores the next character (usually the newline)
    
    return 0;
}
Terminal
$ g++ input.cpp -o input
$ ./input

You entered: 25
Enter two numbers separated by space: 10 20
Sum: 30
Enter your first name: John

Reading Strings with getline()

The std::cin >> operator has limitations when reading strings that contain spaces. For reading entire lines of text, we use the std::getline() function.

getline() Function

  • std::getline(std::cin, string_variable): Reads an entire line into a string variable
  • Reads until a newline character is encountered
  • The newline character is extracted but not stored in the string
  • Can specify a custom delimiter as a third parameter

getline() Examples

#include <iostream>
#include <string>

int main() {
    // Reading a full name with spaces
    std::string fullName;
    std::cout << "Enter your full name: ";
    std::getline(std::cin, fullName);
    std::cout << "Hello, " << fullName << "!" << std::endl;
    
    // Reading multiple lines
    std::string address;
    std::cout << "Enter your address (press Enter twice to finish):" << std::endl;
    std::string line;
    while (std::getline(std::cin, line) && !line.empty()) {
        address += line + "\n";
    }
    std::cout << "Your address:" << std::endl;
    std::cout << address << std::endl;
    
    // Using a custom delimiter
    std::string csvData;
    std::cout << "Enter comma-separated values: ";
    std::getline(std::cin, csvData, ',');
    std::cout << "First value: " << csvData << std::endl;
    
    return 0;
}

Mixing cin and getline()

When mixing std::cin >> and std::getline(), you need to be careful about the newline character left in the input buffer. After using std::cin >>, call std::cin.ignore() to clear the buffer before using std::getline().

Error Handling with cin

Input operations can fail for various reasons, such as entering invalid data. C++ provides ways to check and handle input errors.

Checking Input State

  • std::cin.fail(): Returns true if the last input operation failed
  • std::cin.good(): Returns true if the stream is in a good state
  • std::cin.bad(): Returns true if there's a serious error
  • std::cin.eof(): Returns true if end-of-file is reached
  • std::cin.clear(): Clears error flags
  • std::cin.ignore(): Discards characters from the input buffer

Input Error Handling Examples

#include <iostream>
#include <limits>  // For std::numeric_limits

int main() {
    int number;
    
    std::cout << "Enter an integer: ";
    
    // Check if input is valid
    while (!(std::cin >> number)) {
        std::cout << "Invalid input. Please enter an integer: ";
        
        // Clear the error flag
        std::cin.clear();
        
        // Discard the rest of the line
        std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
    }
    
    std::cout << "You entered: " << number << std::endl;
    
    // Example with range validation
    int age;
    do {
        std::cout << "Enter your age (1-120): ";
        
        while (!(std::cin >> age)) {
            std::cout << "Invalid input. Please enter a number: ";
            std::cin.clear();
            std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
        }
        
        if (age < 1 || age > 120) {
            std::cout << "Age must be between 1 and 120." << std::endl;
        }
    } while (age < 1 || age > 120);
    
    std::cout << "Your age is: " << age << std::endl;
    
    return 0;
}

Manipulators and Formatting

C++ provides manipulators that can be used with I/O streams to format input and output. These manipulators are defined in the <iomanip> header.

Common Output Manipulators

  • std::setw(n): Sets the field width to n characters
  • std::setprecision(n): Sets the precision for floating-point values
  • std::fixed: Uses fixed-point notation for floating-point values
  • std::scientific: Uses scientific notation for floating-point values
  • std::setfill(c): Sets the fill character to c
  • std::left: Left-aligns the output
  • std::right: Right-aligns the output
  • std::boolalpha: Outputs boolean values as "true" or "false"

Output Formatting Examples

#include <iostream>
#include <iomanip>
#include <string>

int main() {
    // Setting width and fill character
    std::cout << std::setw(20) << std::setfill('*') << "Hello" << std::endl;
    std::cout << std::setw(20) << std::setfill('-') << "World" << std::endl;
    
    // Floating-point formatting
    double pi = 3.14159265359;
    std::cout << "Default: " << pi << std::endl;
    std::cout << "Fixed: " << std::fixed << pi << std::endl;
    std::cout << "Scientific: " << std::scientific << pi << std::endl;
    std::cout << "Precision 2: " << std::setprecision(2) << pi << std::endl;
    std::cout << "Precision 5: " << std::setprecision(5) << pi << std::endl;
    
    // Reset to default
    std::cout << std::defaultfloat;
    
    // Boolean formatting
    bool flag = true;
    std::cout << "Default bool: " << flag << std::endl;
    std::cout << "Boolalpha: " << std::boolalpha << flag << std::endl;
    
    // Alignment
    std::cout << std::left << std::setw(15) << "Left" << "|" << std::endl;
    std::cout << std::right << std::setw(15) << "Right" << "|" << std::endl;
    
    // Creating a formatted table
    std::cout << "\nFormatted Table:\n";
    std::cout << std::left << std::setw(15) << "Name" 
              << std::setw(10) << "Age" 
              << std::setw(10) << "Score" << std::endl;
    std::cout << std::string(35, '-') << std::endl;
    std::cout << std::left << std::setw(15) << "Alice" 
              << std::setw(10) << 25 
              << std::setw(10) << 95.5 << std::endl;
    std::cout << std::left << std::setw(15) << "Bob" 
              << std::setw(10) << 30 
              << std::setw(10) << 87.2 << std::endl;
    
    return 0;
}

Understanding the Compilation Process

The compilation process transforms human-readable C++ source code into machine-readable executable code. This process involves several stages, each with a specific purpose.

Stages of Compilation

  1. Preprocessing: Handles preprocessor directives (#include, #define, etc.)
  2. Compilation: Translates preprocessed code into assembly code
  3. Assembly: Converts assembly code into machine code (object files)
  4. Linking: Combines object files and libraries into an executable

Compilation Process Visualization

Source Code (program.cpp)
        |
        v
Preprocessing (program.i)
        |
        v
Compilation (program.s)
        |
        v
Assembly (program.o)
        |
        v
Linking (program.exe/program.out)
        |
        v
Executable Program

Preprocessing Stage

The preprocessing stage is the first step in the compilation process. The preprocessor handles all directives that begin with a hash symbol (#).

Preprocessor Tasks

  • File Inclusion: Replaces #include directives with the contents of the specified files
  • Macro Expansion: Replaces macros defined with #define
  • Conditional Compilation: Processes #ifdef, #ifndef, #if, #else, #elif, and #endif directives
  • Comment Removal: Removes all comments from the source code

Preprocessing Example

// Original source code (program.cpp)
#include <iostream>
#define PI 3.14159
#define SQUARE(x) ((x) * (x))

int main() {
    double radius = 5.0;
    double area = PI * SQUARE(radius);
    std::cout << "Area: " << area << std::endl;
    return 0;
}

// After preprocessing (program.i)
// Contents of iostream included here (thousands of lines)
// ...

int main() {
    double radius = 5.0;
    double area = 3.14159 * ((radius) * (radius));
    std::cout << "Area: " << area << std::endl;
    return 0;
}

Viewing Preprocessed Output

You can see the preprocessed output using the -E flag with g++: 

g++ -E program.cpp -o program.i
This can be useful for debugging macro expansions and understanding how includes work.

Compilation Stage

The compilation stage takes the preprocessed code and translates it into assembly language, which is a low-level representation of the code that's specific to the target architecture.

Compiler Tasks

  • Lexical Analysis: Breaks the code into tokens (keywords, identifiers, operators, etc.)
  • Syntax Analysis: Checks if the code follows the grammar rules of C++
  • Semantic Analysis: Checks for semantic errors (type mismatches, undeclared variables, etc.)
  • Optimization: Applies various optimizations to improve performance
  • Code Generation: Generates assembly code from the analyzed and optimized code

Assembly Code Example

// C++ code
int add(int a, int b) {
    return a + b;
}

// Generated assembly code (x86-64)
add:
    pushq   %rbp
    movq    %rsp, %rbp
    movl    %edi, -4(%rbp)
    movl    %esi, -8(%rbp)
    movl    -4(%rbp), %eax
    addl    -8(%rbp), %eax
    popq    %rbp
    ret

Viewing Assembly Output

You can see the assembly output using the -S flag with g++: 

g++ -S program.cpp -o program.s
This can be useful for performance optimization and understanding how your code translates to machine instructions.

Assembly Stage

The assembly stage converts the assembly code into machine code, creating object files. These object files contain binary code but are not yet executable.

Object Files

  • Contain machine code for the compiled source
  • Include information for the linker (symbols, relocation entries, etc.)
  • Have extensions like .o (Linux/macOS) or .obj (Windows)
  • Cannot be executed directly because they may reference external symbols

Creating Object Files

// Compile to object file only
g++ -c program.cpp -o program.o

// View object file information
objdump -t program.o  // Show symbols
objdump -d program.o  // Disassemble code
nm program.o          // List symbols

Linking Stage

The linking stage is the final step in creating an executable. The linker combines one or more object files and libraries into a single executable program.

Linker Tasks

  • Symbol Resolution: Resolves references to external symbols (functions and variables)
  • Relocation: Adjusts code and data addresses to match the final memory layout
  • Library Inclusion: Includes code from standard and user-defined libraries
  • Address Assignment: Assigns final memory addresses to code and data

Linking Example

// Compile and link in one step
g++ program.cpp -o program

// Compile to object files first, then link
g++ -c main.cpp -o main.o
g++ -c utils.cpp -o utils.o
g++ main.o utils.o -o program

// Link with external libraries
g++ program.cpp -o program -lm  // Link with math library
g++ program.cpp -o program -lpthread  // Link with pthread library

Common Linking Errors

  • Undefined Reference: Occurs when a function or variable is declared but not defined
  • Multiple Definition: Occurs when the same symbol is defined in multiple object files
  • Missing Libraries: Occurs when required libraries are not specified during linking

Common Compiler Flags

Compilers like g++ provide various flags to control the compilation process. Understanding these flags is essential for effective C++ development.

Essential g++ Flags

  • -o filename: Specifies the output file name
  • -c: Compile only, do not link (creates object file)
  • -E: Preprocess only, do not compile
  • -S: Compile only, do not assemble (creates assembly file)
  • -g: Include debug information
  • -O0, -O1, -O2, -O3: Optimization levels (0 = no optimization, 3 = maximum optimization)
  • -Wall: Enable all warning messages
  • -Werror: Treat warnings as errors
  • -std=c++11, -std=c++14, -std=c++17, -std=c++20: Specify C++ standard
  • -I directory: Add directory to include search path
  • -L directory: Add directory to library search path
  • -l library: Link with library

Compiler Flag Examples

// Basic compilation
g++ program.cpp -o program

// With debug information
g++ -g program.cpp -o program

// With optimization level 2
g++ -O2 program.cpp -o program

// With all warnings and treat warnings as errors
g++ -Wall -Werror program.cpp -o program

// Using C++20 standard
g++ -std=c++20 program.cpp -o program

// With custom include directory
g++ -I./include program.cpp -o program

// With custom library directory and library
g++ -L./lib program.cpp -o program -lmylib

Build Systems and Makefiles

For larger projects with multiple source files, manually compiling each file becomes tedious. Build systems like Make automate this process.

Makefile Basics

A Makefile contains rules that specify how to build targets (usually executables or object files) from dependencies (source files). Each rule has the format:

target: dependencies
    command

Simple Makefile Example

# Makefile for a simple C++ project

# Variables
CXX = g++
CXXFLAGS = -Wall -std=c++11
TARGET = program
SRCS = main.cpp utils.cpp
OBJS = $(SRCS:.cpp=.o)

# Default target
all: $(TARGET)

# Link the executable
$(TARGET): $(OBJS)
    $(CXX) $(CXXFLAGS) -o $(TARGET) $(OBJS)

# Compile source files into object files
%.o: %.cpp
    $(CXX) $(CXXFLAGS) -c $< -o $@

# Clean up build artifacts
clean:
    rm -f $(OBJS) $(TARGET)

# Phony targets (targets that don't represent files)
.PHONY: all clean

Using the Makefile

make          # Build the program
make clean    # Remove object files and executable
make all      # Same as 'make'

Create a program that accepts command-line arguments and:

  4. If the first argument is "--version", prints a version number
#include <iostream>
#include <string>

void printHelp() {
    std::cout << "Usage: program [options]" << std::endl;
    std::cout << "Options:" << std::endl;
    std::cout << "  --help     Show this help message" << std::endl;
    std::cout << "  --version  Show version information" << std::endl;

    std::cout << "Program Version 1.0.0" << std::endl;
    std::cout << "Compiled on " << __DATE__ << " at " << __TIME__ << std::endl;

int main(int argc, char* argv[]) {
    // Check for help or version flags
    if (argc > 1) {
        std::string firstArg = argv[1];
        if (firstArg == "--help") {

        } else if (firstArg == "--version") {
            printVersion();
            return 0;
        }
    }

    std::cout << "Number of arguments: " << argc << std::endl;
    
    // Print each argument
    std::cout << "Arguments:" << std::endl;
    for (int i = 0; i < argc; ++i) {
        std::cout << "  " << i << ": " << argv[i] << std::endl;
    }
    
    return 0;
}

Exercise 6: Input Validation

Write a program that asks the user for a number between 1 and 100 and:

  1. Validates that the input is a valid integer
  2. Validates that the number is within the specified range

  4. Repeats the prompt until valid input is received
  5. Prints a success message with the validated number
#include <iostream>
#include <limits>

    int number;
    bool validInput = false;
    
    while (!validInput) {
        std::cout << "Enter a number between 1 and 100: ";

        if (!(std::cin >> number)) {
            std::cout << "Error: Invalid input. Please enter a number." << std::endl;

        if (number < 1 || number > 100) {
            std::cout << "Error: Number must be between 1 and 100." << std::endl;
        } else {
            validInput = true;
        }
    }
    
    // Print success message
    std::cout << "Success! You entered the number " << number << "." << std::endl;
    
    return 0;
}

Exercise 8: Advanced Input/Output

Create a program that:

  1. Asks the user for their personal information:
    • Full name (may contain spaces)
    • Age (integer)
    • Height in meters (double)
    • Weight in kilograms (double)
  2. Calculates and displays their BMI (Body Mass Index) using the formula: BMI = weight / (height * height)
  3. Displays the BMI category:
    • Underweight: BMI < 18.5
    • Normal weight: 18.5 ≤ BMI < 25
    • Overweight: 25 ≤ BMI < 30
    • Obese: BMI ≥ 30
  4. Formats all output neatly with appropriate labels
#include <iostream>
#include <iomanip>
#include <string>
#include <limits>

int main() {
    std::string fullName;
    int age;
    double height, weight;
    
    // Get user input
    std::cout << "=== Personal Information ===" << std::endl;
    
    std::cout << "Enter your full name: ";
    std::getline(std::cin, fullName);
    
    std::cout << "Enter your age: ";
    while (!(std::cin >> age) || age <= 0) {
        std::cout << "Invalid age. Please enter a positive integer: ";
        std::cin.clear();
        std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
    }
    
    std::cout << "Enter your height in meters: ";
    while (!(std::cin >> height) || height <= 0) {
        std::cout << "Invalid height. Please enter a positive number: ";
        std::cin.clear();
        std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
    }
    
    std::cout << "Enter your weight in kilograms: ";
    while (!(std::cin >> weight) || weight <= 0) {
        std::cout << "Invalid weight. Please enter a positive number: ";
        std::cin.clear();
        std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
    }
    
    // Calculate BMI
    double bmi = weight / (height * height);
    
    // Determine BMI category
    std::string category;
    if (bmi < 18.5) {
        category = "Underweight";
    } else if (bmi < 25) {
        category = "Normal weight";
    } else if (bmi < 30) {
        category = "Overweight";
    } else {
        category = "Obese";
    }
    
    // Display results
    std::cout << "\n=== Results ===" << std::endl;
    std::cout << std::left << std::setw(20) << "Name:" << fullName << std::endl;
    std::cout << std::left << std::setw(20) << "Age:" << age << " years" << std::endl;
    std::cout << std::left << std::setw(20) << "Height:" << std::fixed << std::setprecision(2) << height << " m" << std::endl;
    std::cout << std::left << std::setw(20) << "Weight:" << std::fixed << std::setprecision(2) << weight << " kg" << std::endl;
    std::cout << std::left << std::setw(20) << "BMI:" << std::fixed << std::setprecision(1) << bmi << std::endl;
    std::cout << std::left << std::setw(20) << "Category:" << category << std::endl;
    
    return 0;
}
