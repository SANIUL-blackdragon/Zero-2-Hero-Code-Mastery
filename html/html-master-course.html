Introduction to HTML

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a web page using elements and tags. HTML is the foundation of all websites and web applications.

üí° Fun Fact: HTML was created by Tim Berners-Lee in 1991 and has evolved significantly since then. The latest version is HTML5, which introduced many new features and capabilities.

  • Creative Expression: Build your own websites and bring your ideas to life

HTML Basics

HTML Document Structure

Every HTML document follows a basic structure. Here's the minimal HTML5 document:

Basic HTML5 Document
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First HTML Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>
</body>
</html>

Document Structure Explained:

  • <!DOCTYPE html> - Defines the document type and HTML version (HTML5)
  • <html> - Root element of the HTML page
  • <head> - Contains meta-information about the document
  • <body> - Contains the visible page content

HTML Elements and Tags

HTML elements are defined by tags, which are keywords surrounded by angle brackets. Most elements have an opening tag and a closing tag:

HTML Element Structure
<tagname>Content goes here</tagname>

Types of Elements:

  • Block-level elements: Take up the full width available (e.g., <div>, <p>, <h1>)
  • Inline elements: Take up only as much width as necessary (e.g., <span>, <a>, <strong>)
  • Self-closing elements: Don't need a closing tag (e.g., <img>, <br>, <hr>)

HTML Attributes

Attributes provide additional information about HTML elements. They are always specified in the start tag and usually come in name/value pairs.

HTML Attributes

What is Semantic HTML?

Semantic HTML uses HTML elements according to their intended purpose. It uses elements that clearly describe their meaning to both the browser and the developer.

üí° Why Semantic HTML Matters:

Benefits of Semantic HTML

Accessibility

Semantic HTML greatly improves accessibility for users with disabilities:

  • Screen Readers: Screen readers use semantic elements to understand page structure and navigate content
  • Keyboard Navigation: Proper semantic structure improves keyboard navigation
  • ARIA Roles: Many semantic elements have implicit ARIA roles

SEO Benefits

Search engines benefit from semantic HTML:

  • Better Understanding: Search engines can better understand content structure and context
  • Rich Snippets: Semantic markup can lead to rich search results
  • Content Hierarchy: Proper heading structure helps with content organization

Code Maintainability

Semantic HTML makes code easier to maintain:

  • Readability: Code is more descriptive and easier to understand
  • Collaboration: Team members can quickly understand the structure
  • Debugging: Easier to identify and fix issues

Future-Proofing

Semantic HTML ensures better compatibility with future technologies:

  • New Devices: Better support for new devices and browsers
  • Emerging Technologies: Compatibility with AI, voice assistants, etc.
  • Web Standards: Adherence to current and future web standards
‚ö†Ô∏è Common Mistakes to Avoid:
  • Using <div> for everything instead of semantic elements
  • Skipping heading levels (e.g., from <h1> to <h3>)
  • Not using <label> elements with form controls
  • Using tables for layout instead of data presentation
  • Nesting block-level elements inside inline elements
üí° Best Practices:
  • Always choose the most semantic element for your content
  • Maintain proper heading hierarchy throughout your document
  • Use <main> only once per page for the main content
  • Provide alternative text for all meaningful images
  • Test your pages with screen readers and accessibility tools

What is Web Accessibility?

Web accessibility means that websites, tools, and technologies are designed and developed so that people with disabilities can use them. This includes people with auditory, cognitive, neurological, physical, speech, and visual disabilities.

üí° Why Accessibility Matters:
  • Inclusive Design: 15% of the world's population lives with some form of disability
  • Legal Requirements: Many countries have laws requiring web accessibility
  • Better UX: Accessible sites are easier to use for everyone
  • SEO Benefits: Search engines favor accessible websites
  • Broader Audience: Reach more users including those with temporary disabilities

WCAG Guidelines

The Web Content Accessibility Guidelines (WCAG) provide recommendations for making web content more accessible. WCAG 2.1 is organized around four principles:

1. Perceivable

Information and user interface components must be presentable to users in ways they can perceive.

Guideline Description HTML Implementation
Text Alternatives Provide text alternatives for non-text content alt, aria-label, aria-labelledby
Time-based Media Provide alternatives for time-based media <track>, transcripts, captions
Adaptable Create content that can be presented in different ways Semantic HTML, proper heading structure
Distinguishable Make it easier for users to see and hear content Color contrast, sufficient font size

2. Operable

User interface components and navigation must be operable.

Guideline Description HTML Implementation
Keyboard Accessible Make all functionality available from a keyboard Tab order, tabindex, keyboard events
Enough Time Provide users enough time to read and use content Controls for time limits, animations
Seizures Do not design content that causes seizures Avoid flashing content, provide controls
Navigable Provide ways to help users navigate and find content Skip links, landmarks, breadcrumbs

3. Understandable

Information and the operation of user interface must be understandable.

Guideline Description HTML Implementation
Readable Make text content readable and understandable lang attribute, clear language
Predictable Make Web pages appear and operate in predictable ways Consistent navigation, clear form labels
Input Assistance Help users avoid and correct mistakes Form validation, error messages, instructions

4. Robust

Content must be robust enough that it can be interpreted reliably by a wide variety of user agents, including assistive technologies.

Guideline Description HTML Implementation
Compatible Maximize compatibility with current and future user agents Valid HTML, proper ARIA usage
